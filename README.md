Built with attention to scalability, simplicity for non-technical users, and automation-first workflows.

## Engineering philosophy

> I prefer shipping over endless architecture discussions.

Good systems should survive changing requirements, scale with small teams, stay understandable after years, and automate repetitive work whenever possible.

I value **practical engineering** more than trendy stacks.
